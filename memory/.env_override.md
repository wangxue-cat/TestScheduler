# 环境解析规则

## 三级优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| **1（最高）** | 用户显式指定 | 用户在当前对话中明确说"STG1/STG2/STG3"，直接使用 |
| **2** | 本地缓存 | 上次切换的环境，存储于 `d:\TestScheduler\memory\.env_override.md` |
| **3（兜底）** | testmind 查询 | `testmind:get-current-week-sprint-env` 查 QOA 数据库当前迭代版本环境 |

## 缓存规则

- **每次切换环境都更新缓存**：无论是用户指定还是 testmind 查询到的新环境，都写入本地缓存
- **缓存文件**：`d:\TestScheduler\memory\.env_override.md`
- **缓存更新时机**：
  - 用户说"切到 STG3" → 缓存 STG3
  - 本地缓存丢失，查 testmind 得到 STG2 → 缓存 STG2

## 示例

```
用户说"查 aps.order_info" → 未指定环境 → 读本地缓存 → STG3 → 用 STG3
用户说"STG1 查 aps.order_info" → 指定 STG1 → 用 STG1，更新缓存为 STG1
本地缓存丢失 → 调 testmind → 返回 STG2 → 用 STG2，写入缓存
```
