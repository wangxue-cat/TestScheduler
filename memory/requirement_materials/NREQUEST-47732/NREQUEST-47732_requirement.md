## 需求背景

API 渠道开始是使用 360JIETIAO 产品经营。增量交易已全部走 360API；小部分存量JIETIAO用户静默 360API 额度失败，这部分静默失败的主要是用户状态为非有效的（如注销、手机号二次放号等）。

因未能完全切到 360API，研发、测试需要维护两套产品。因此需要确定此类用户的自动处理流程，降低维护成本。

## 需求内容

1、对静默360API 失败的 JIETIAO 用户，APS 做交易拒绝处理。内部保存交易拒绝原因为：用户状态异常，静默 360API 失败。
