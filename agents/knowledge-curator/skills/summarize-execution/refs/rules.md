# summarize-execution 详细规则

## 1. 去重

已有类似条目只标记不重复创建。去重来源：
- `testmind:knowledge-retrieve` 语义检索
- 本地 `memory/knowledge/` 目录对比

## 2. 条目最低要求

每个条目至少包含：类型、标题、内容、关联需求。

## 3. 入库确认

知识条目草稿需用户确认后才通过 `curate-entry` 入库。

## 4. Skill 补丁性质

Skill 补丁为建议性质，不自动修改 Skill 文件，需用户单独审批。

## 5. 补丁合并

同 Skill 同类型的补丁合并输出，避免碎片化。

## 6. 补丁定位

补丁必须指向文件中的具体位置（章节/步骤编号），不能只写"改一下XX"。

## 知识提取维度

| 维度 | 分析方法 |
|------|---------|
| **接口经验** | 新接口的调用方式和参数特征、特殊错误码、隐式依赖 |
| **踩坑记录** | 参数组装问题、环境差异、时序问题 |
| **参数模式** | 参数组合导致特定结果、边界值 |
| **失败模式** | 常见失败类型和根因、与历史 Bug 关联 |

## Skill 改进分析维度

| 维度 | 分析方法 | 示例 |
|------|---------|------|
| **遗漏步骤** | 执行中实际做了但 Skill 未定义的步骤 | OB表验证前等5秒 |
| **参数坑点** | 参数组装中遇到但 Skill 未记载的问题 | repay_order_id 字段映射 |
| **环境差异** | 不同环境的行为差异 | Apollo 超时但生效 |
| **验证盲区** | 验证方法缺陷导致误判 | 仅靠接口返回判断路由 |
| **过时信息** | Skill 中步骤因代码变更不再适用 | 旧版参数格式废弃 |

### 补丁置信度

- ≥3 次同类执行佐证 → `high`
- 首次发现 → `medium`
- 推测 → `low`
