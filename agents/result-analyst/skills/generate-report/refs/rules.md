# generate-report 详细规则

## 1. 产出目录

报告必须生成到 `memory/test_results/reports/` 目录。

## 2. 失败用例展示

失败用例必须展示完整的请求体和响应体，包含所有字段。

## 3. HTML 自包含

HTML 报告必须自包含（无外部 CSS/JS 依赖）。

## 4. 文件命名

报告文件名格式：`{requirement_id}_execution_report.{md|html}`

## 5. Markdown 报告章节

1. **执行概览** — 总数/通过/失败/通过率
2. **需求信息** — 需求ID、标题、迭代版本、测试环境
3. **通过用例** — 列表（case名称、步骤数、耗时）
4. **失败用例** — 详细展示（步骤、请求、响应、预期vs实际、日志片段）
5. **未完成用例** — 执行到哪步、中断原因
6. **根因分析** — 来自 failure_analysis（如有）
7. **Bug 清单** — 已提交/待提交的 Bug
8. **结论与建议** — 是否建议发版、风险点

## 6. HTML 报告要素

- 渐变色头部（需求ID、迭代、环境、日期）
- 汇总卡片（总数/通过/失败/通过率）
- 可展开的 case 卡片（按通过/失败着色）
- 请求/响应代码高亮
- 校验结果勾/叉标记
- Bug 链接
