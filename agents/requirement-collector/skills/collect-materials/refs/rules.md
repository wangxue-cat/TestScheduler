# collect-materials 详细规则

## 1. 关键材料缺失判定

需求文档内容为空 **且** 测试人员说明内容为空时，触发以下提示：

```
⚠️ 关键材料缺失
缺少以下材料：
  - 需求文档（未获取到需求描述正文）
  - 测试人员说明（测试人员未提供补充说明）

缺少这些材料可能导致用例覆盖不全或遗漏关键场景。
是否继续编写用例？[继续 / 中止并补充材料]
```

- 仅需求文档缺失（测试说明有内容）→ 不提示，继续
- 仅测试说明缺失（需求文档有内容）→ 不提示，继续
- 两者都缺失 → **必须提示用户确认**

## 2. 非关键材料

开发文档、代码仓库缺失不阻塞流程，直接跳过。

## 3. 已有目录处理

已有材料目录时，询问用户是否覆盖更新。

## 4. 编码规范

所有文件使用 UTF-8 编码。

## 5. 材料完整性检查

| 材料 | 是否关键 | 缺失时行为 |
|------|---------|-----------|
| 需求文档 | ✅ 关键 | 提示用户 |
| 测试人员说明 | ✅ 关键 | 提示用户 |
| 开发文档 | ❌ 非关键 | 静默跳过 |
| 代码仓库 | ❌ 非关键 | 静默跳过 |
